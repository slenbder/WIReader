# WIReader Product Manifesto

Version: 1.0

This document defines the long-term philosophy of WIReader. It should change only when the product vision itself changes, not when
features or implementation details evolve.

---

# Why WIReader Exists

WIReader was not created because the world needed another EPUB reader.

It was created because reading gradually became harder than it should be.

Not because books changed.

Because everything around them did.

Modern digital products compete for attention. Notifications, recommendations, feeds, short videos and endless interactions constantly interrupt the natural process of reading.

Reading deserves a different environment.

WIReader exists to create that environment.

---

# Mission

Remove everything that stands between a person and a book.

Not convince people to read.

Not gamify reading.

Not optimize reading.

Simply make reading feel natural again.

---

# Vision

Reading should become a seamless part of everyday life.

Opening a book should require no more effort than opening a favorite conversation or music app.

The application should fit into life instead of asking life to adapt to it.

---

# Core Belief

Reading is not an event.

Reading is part of life.

---

# Philosophy

Every element inside WIReader exists for one reason:

To help the user spend less time navigating the application and more time inside the book.

The interface is not the product.

The reading experience is the product.

When the user starts reading, the interface has already completed its job.

---

# What WIReader Is

WIReader is a calm place inside a noisy digital world.

It respects the user's attention.

It never competes with the book.

It never asks for attention it does not truly need.

It creates confidence instead of excitement.

It disappears whenever possible.

---

# What WIReader Is Not

WIReader is not a social network.

It is not a marketplace.

It is not a productivity system.

It is not another content platform fighting for engagement.

It is not a collection of features.

Every feature must justify its existence by improving the reading experience.

---

# Design Philosophy

Beauty comes from clarity.

Motion exists to explain, never to impress.

Materials support the content, never compete with it.

Complexity is revealed only when the user asks for it.

Silence is a design tool.

Space is a design tool.

Restraint is a design tool.

---

# Product Principles

The book is always the primary interface.

The fastest path should always lead to reading.

Frequently used actions should always be easier to reach than advanced functionality.

The interface should feel predictable.

Every interaction should feel intentional.

Every animation should have purpose.

Every screen should earn its complexity.

---

# The North Star

Every product decision should reduce the time spent inside the interface and increase the time spent inside the book.

If a decision does not help achieve this, it does not belong in WIReader.

---

# Long-Term Goal

The greatest compliment WIReader can receive is not:

"This is the most beautiful reading app."

It is:

"I started reading again."

Because if people return to books, the application has fulfilled its purpose.
